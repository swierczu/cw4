Git merge – krótki opis

Komenda git merge służy do łączenia zmian z jednej gałęzi (branch) do drugiej. Najczęściej używa się jej do scalania feature branchy z gałęzią główną master lub main.

Proces łączenia odbywa się przez integrację historii commitów. Jeśli zmiany nie kolidują ze sobą, Git automatycznie scala je w jeden wspólny commit. W przypadku konfliktów, użytkownik musi ręcznie rozwiązać niezgodności w plikach.

Przykładowe użycie:

git checkout master
git merge feature_branch

Powyższe polecenia przełączają się na gałąź master i scalają w nią zmiany z feature_branch.
