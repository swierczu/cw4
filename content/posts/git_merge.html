Jak działa komenda git merge

Co to jest git merge?

Komenda git merge służy do łączenia zmian z jednego brancha Git z innym. Najczęściej używamy jej, aby scalić zmiany z brancha funkcyjnego (feature branch) do brancha głównego (np. main lub master).

Jak działa git merge?

  • git merge <branch> pobiera zmiany z wybranego brancha i próbuje je połączyć z aktualnie aktywnym branchem.
  • Jeśli zmiany nie kolidują ze sobą, merge przebiega automatycznie.
  • W przypadku konfliktów Git prosi o ręczne rozwiązanie różnic w plikach.
  • Po poprawnym rozwiązaniu konfliktów i zatwierdzeniu zmian, historia commitów zawiera połączone zmiany z obu gałęzi.

Przykład użycia

git checkout main
  git merge feature_branch
