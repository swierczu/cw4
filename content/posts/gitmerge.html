Jak działa komenda git merge?

Komenda git merge służy do łączenia dwóch lub więcej gałęzi (branchy) w Git. Pozwala ona połączyć zmiany wprowadzone w różnych gałęziach, scalając je w jedną wspólną historię. Najczęściej używa się jej, gdy chcemy zintegrować zmiany z gałęzi roboczej (np. feature) do głównej gałęzi (np. main).

Jak działa git merge?

  • Git porównuje historię aktualnej gałęzi i gałęzi, którą chcemy połączyć.
  • Próbuje automatycznie scalić zmiany.
  • Jeśli zmiany nie kolidują, scalanie zakończy się automatycznie.
  • W przypadku konfliktów (np. zmiany w tych samych liniach plików), Git poprosi o ręczne rozwiązanie konfliktów.

Przykład użycia

git checkout main       # Przełącz się na gałąź główną
git merge feature       # Połącz gałąź 'feature' z 'main'
