Co robi git merge?

Komenda git merge służy do połączenia zmian z jednej gałęzi (branch) z inną. Najczęściej używana jest do łączenia gałęzi roboczych (np. feature) z główną gałęzią projektu (np. main lub master).

Podstawowe użycie

$ git checkout main
$ git merge feature-branch

W tym przykładzie przechodzimy na gałąź main i łączymy do niej zmiany z gałęzi feature-branch.

Typowe zastosowania

  • Łączenie zakończonych funkcjonalności do głównej gałęzi
  • Aktualizacja bieżącej gałęzi o zmiany z innej
  • Współpraca z innymi programistami nad tym samym repozytorium

Uwaga na konflikty

Podczas łączenia może dojść do konfliktów, jeśli zmiany w tych samych fragmentach pliku pojawiły się na obu gałęziach. W takim przypadku Git poprosi o ręczne rozwiązanie konfliktów.
